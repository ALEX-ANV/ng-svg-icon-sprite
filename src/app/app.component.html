{{ title }}

Angular package that provides both a solution for generating sprites and a component for including them.

The Basics (reference, width and styling)

To use if you want to display the icons in original size. Just match the [width] and [height] with the width/height of the equivalent SVG properties.

  • <svg-icon-sprite [src]="'assets/sprites/sprite.svg#star_100x100'" [width]="'100px'"></svg-icon>

.icon-example {{ '{' }}
  color: darkred;
{{ '}' }}
  • <svg-icon-sprite [src]="'assets/sprites/sprite.svg#delete_70x70'" [width]="'70px'" [height]="'65px'"></svg-icon>
  • <svg-icon-sprite [src]="'assets/sprites/sprite.svg#star_100x100'" [classes]="'css-styled-icon'" [width]="'100px'"></svg-icon>

.container ::ng-deep svg.css-styled-icon {{ '{' }}
  use {{ '{' }}
    fill: orange;
    stroke: black;
  {{ '}' }}
{{ '}' }}

Scaling (using CSS transform)

Use this pattern if you want to change the size of the original SVG. Apply same rules as above, wrap the SVG into a div and size it using CSS transforms.

  • .icon-scale-down {{ '{' }}
  // Scale down by 50%
  transform: scale(0.5);
{{ '}' }}
      
  • .icon-scale-up {{ '{' }}
  // Scale up by 400%
  transform: scale(4);
{{ '}' }}

Scaling (using viewBox)

This is an alternative to the CSS scaling method presented above. Scaling via viewBox is less consistent across browsers. The pattern depends on your SVG source file, whether it already contains a viewBox or not. The following example SVGs with an original size of 100x100px are scaled down by 50%. 
  • <!-- SVG source lacks a viewBox -->
<svg-icon-sprite [src]="'assets/sprites/sprite.svg#star_100x100'" [width]="'100px'" [viewBox]="'-50 -50 200 200'"></svg-icon>
      
  • <!-- SVG source contains a viewBox -->
<svg-icon-sprite [src]="'assets/sprites/sprite.svg#star_100x100_viewbox'" [width]="'50px'" [viewBox]="'0 0 100 100'"></svg-icon>

Manipulation (using HTML properties)

  • <svg-icon-sprite [src]="'assets/sprites/sprite.svg#' + icon" [width]="width"></svg-icon>

Adding Custom SVG Attributes

  • <svg-icon-sprite [src]="'assets/sprites/sprite.svg#delete_70x70'" [attribute]="['focusable', 'false']"></svg-icon>
Adding focusable="false" will prevent browser focus on SVG element in IE11.

Multicolor Icons

If you want to include an multicolor icon, you will have to provide it form a separate sprite file (where the fill and stroke attributes are not stripped). Remember that you will not be able to overwrite styles via CSS. 
  • <svg-icon-sprite [src]="'assets/sprites/image-sprite.svg#multicolor-image'"></svg-icon>
